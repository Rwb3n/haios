## Root Causes

1. **High threshold (0.85)** — strict similarity means most concepts find only one near-neighbor
2. **Greedy single-pass** — seeds from one concept, sweeps remaining; no iterative refinement
3. **No centroid re-computation** — cluster centroid is just the first item, not updated as members join
4. **TODO in code acknowledged**: `_build_clusters` has comment "Consider more sophisticated clustering (DBSCAN, etc.)"

## Why It Matters for E3

Pair-only synthesis is shallow — it merges two near-duplicates rather than discovering thematic clusters across multiple sources. Richer clusters (5-20 members) produce higher-order insights that connect disparate knowledge.

## Potential E3 Work

- Replace greedy with DBSCAN or hierarchical agglomerative clustering
- Lower threshold to ~0.75 for initial grouping, use LLM to validate coherence
- Implement centroid re-computation as members join
- Two-pass approach: broad clusters first, then LLM-refined sub-clusters
- Metric: target median cluster size of 4-5 (vs current 2)
