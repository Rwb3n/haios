# Observation: Tests Import lib/ Instead of modules/ After E2-279 Decomposition

**Finding:** E2-279 module decomposition created new modules (cascade_engine.py, backfill_engine.py, spawn_tree.py) but tests still import from legacy lib paths:
- test_lib_cascade.py: `from cascade import` (should import cascade_engine)
- test_backfill.py: `from backfill import` (should import backfill_engine)
- test_lib_spawn.py: `from spawn import` (should import spawn_tree)

**Implication:** Tests pass because lib implementations still exist, but module-specific code paths are untested. This creates false confidence in test coverage.

**Potential Action:** Update test imports to reference modules/ instead of lib/, or deprecate the lib implementations if modules are now authoritative.
