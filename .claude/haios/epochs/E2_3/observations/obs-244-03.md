# Observation: Scaffold Command Argument Parsing Broken

## What Was Observed

During observation triage, creating work items via `just scaffold work_item "title" --type X --chapter Y --arc Z` failed. The command treated `--type investigation` as part of the title string instead of as a flag.

## Evidence

```
> just scaffold work_item "Node Transitions Value Assessment" --type investigation --chapter CH-006 --arc workuniversal
Created: docs/work/active/Node Transitions Value Assessment/WORK.md

# Resulting WORK.md:
title: "--type investigation --chapter CH-006 --arc workuniversal"
type: {{TYPE}}
chapter: null
```

## Impact

- AgentUX friction: must scaffold then manually edit 3+ fields
- Error-prone: easy to forget a field
- 7 work items required 14 operations instead of 7

## Root Cause

cli.py scaffold command uses positional args only: `scaffold <template> <backlog_id> <title>`
The --flags are not parsed.

## Suggested Fix

Update cli.py scaffold to accept optional flags:
- `--type` → sets type field
- `--chapter` → sets chapter field
- `--arc` → sets arc field
- `--spawned-by` → already exists
