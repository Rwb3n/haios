# Recursive Thinking

Decompose problems into smaller self-similar subproblems. Identify the base case and the reduction step. Recognize when a problem contains itself at a smaller scale.

## When to use
- Complex problems that can be broken into similar sub-pieces
- Hierarchical or nested structures
- When solving a simpler version would illuminate the full solution

## Guiding questions
1. Can this problem be expressed in terms of a smaller version of itself?
2. What is the base case — the simplest version that can be solved directly?
3. What is the reduction step — how does solving the smaller version help?
4. Are there overlapping subproblems (memoization opportunity)?
5. What is the depth of recursion — how many levels deep does this go?

## Output structure
- Problem decomposition
- Base case
- Reduction step
- Depth/complexity estimate
